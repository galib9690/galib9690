I build **AI-Native Hydrology** — an open ecosystem of agent-operable Python packages and an MCP server that lets a general AI agent conduct end-to-end hydrological research with full scientific defensibility as a structural byproduct.

My PhD at Purdue frames this across three pillars: **Diagnose** (what breaks when AI meets hydrology), **Fill the Gaps** (build the missing infrastructure), **Democratize** (open tools anyone can extend).

---

## The AI-Hydro Ecosystem

> [github.com/AI-Hydro](https://github.com/AI-Hydro) · agent-native · Apache 2.0

A layered family of purpose-built packages, all surfaced through `aihydro-tools` as the MCP server meta-package:
